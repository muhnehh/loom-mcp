FAQ

How is LoomMCP different from jCodeMunch?
We use the same tree-sitter approach but we have HIGHER token reduction (97.75% vs 95%) and we're FREE. jCodeMunch charges $79-1,999/year for commercial use. LoomMCP is MIT licensed — free forever.
Which MCP clients does LoomMCP work with?
Claude Code, Claude Desktop, Cursor, Windsurf, Continue, VS Code with MCP extension, and any MCP-compatible client.
Why not just use a larger context window?
Bigger context = higher costs + lower quality. Context window pricing is per-token. Every byte you don't send is money saved. Plus, larger contexts lead to worse modelRecall — the model has more to process.
Can I use LoomMCP and jCodeMunch together?
Technically yes, but there's no point — you'd double your costs without benefit. Choose one.
How much can LoomMCP reduce my token usage?
Our benchmark shows 97.75% token reduction on average. For a typical project with 50K tokens of source code, you'd use ~1,250 tokens instead of 50,000 — a 40x reduction.
